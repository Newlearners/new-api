如何查看分组、价格、模型名称？

以控制台「模型 / 计价」页为准（地址以网站实际菜单为准）。对照方式如下：

  • 分组在哪看：左侧 「可用令牌分组」 里点选（如 GPT、default、vip、临时、svip、opus4.7 等）。选中后，右侧只显示该分组下可用模型。
  • 模型名称是哪一行：右侧每张 模型卡片最上面 的粗体字符串即为模型 ID（新建令牌、填 CC Switch / 客户端时要与这里一致）。
  • 价格与怎么计费：卡片内 「模型价格」 一行即单价说明（常见为「$x.xx / 次」或按量文案）；卡片角上的 小标签 标明 按次计费 或 按量计费。左侧 「计费类型」 可筛选只看某一种计费。

三、常见报错

点击展开

环境冲突 / 登过官方 → 按顺序试。没报错可直接跳过本节。

常见报错：所在地区不可用（选做）

配置完成后若终端出现与下图类似提示（没有则可跳过本小节）：红色报错 Unable to connect to Anthropic services、Failed to connect to api.anthropic.com: ERR_BAD_REQUEST，并提示检查网络或地区；官方说明见 anthropic.com/supported-countries。

原因：缺少完成引导（onboarding）校验。可在用户目录下的 ~/.claude.json（Windows 多为 C:\Users\你的用户名\.claude.json）中加入或合并字段：

"hasCompletedOnboarding": true

确保该键位于 JSON 顶层对象中（与同级其它配置并列）。保存后重启终端再试。

不要加错文件：请改 ~/.claude.json，不要写到 ~/.claude/settings.json。

Claude 桌面 App

官方客户端 · Developer 模式 · Configure third-party inference（Anthropic-compatible 网关）

请先完成首页一、注册账号、额度、令牌，拿到你的 API key。网关 Base URL 示例：https://newapi.celink.top（以控制台为准）。模型 ID 须与令牌分组一致，勿填错。

需桌面版 Claude（Windows / macOS）。以下为「三方推理」链路，不是 Claude Code 终端。

01 打开应用，停留在登录页

启动 Claude 桌面端，出现 Welcome back 等登录界面即可（可用邮箱或 Google 登录）。

02 开启开发者模式

顶部菜单 Help → Troubleshooting → 选择 Enable developer mode。

03 打开「配置第三方推理」

菜单栏出现 Developer 后，点击 Developer → Configure Third-Party Inference…。

04 连接方式与网关凭据

左侧 Connection 选中 Gateway (Anthropic-compatible)。在 Gateway Credentials 中填写：Gateway base URL（如 https://newapi.celink.top）、Gateway API key 填你自己的令牌、Gateway auth scheme 选 bearer。下方 Identity & Models 里点 + Add 添加要用的模型 ID。

05 模型列表与保存

在 Model list 中逐项填写模型 ID（如 claude-opus-4-6、claude-sonnet-4-6），名称须与渠道一致。Offer 1M-context variant 建议保持关闭：部分逆向/中转渠道不支持 1M 窗口。最后点击 Apply locally 保存。

模型名、分组与令牌须对应；改完后若对话仍异常，可尝试重启 App。

06 对话界面与切换模型

进入会话后，界面类似下图（Cowork 等视图）。右下角可在已配置的模型之间切换（如 Opus 4.6、Sonnet 4.6 等）；左下角显示 Gateway 等与连接方式有关的提示时可对照上文核对配置。
